# Synthetix 桌面测试版 — 安装与测试说明

> 这是 **Windows 测试包**（v0.10.9），用于内部测试。

## 给测试者

### 1. 安装

1. 双击 `Synthetix Setup 0.10.9.exe`。
2. 按向导选择安装路径（默认装到当前用户目录，**无需管理员权限/UAC**）。
3. 勾选"Launch Synthetix now"完成安装。

### 2. 首次启动

应用启动后会**自动**完成以下初始化（无需手动操作）：
- 生成加密密钥（JWT、数据加密）
- 创建本地数据库（SQLite，应用全部 8 个迁移）

首次启动可能比平时慢几秒（建库 + 预热）。之后会自动打开应用窗口。

> 数据全部存在 `%APPDATA%\Synthetix\`（数据库、文档、RAG 索引、密钥），**不在安装目录**。卸载不会删数据。

### 3. 配置 LLM 并跑通主流程

1. 首次进入会要求**创建管理员账号**（本地账号，仅存你机器上）。
2. 在 **设置 → 模型提供商** 配置你的 LLM：
   - 推荐：OpenAI（填 API Key）或 DeepSeek，或任意 OpenAI 兼容服务（Ollama / 本地 vLLM 等）
   - 填好后点"测试连接"确认可用
3. **验证 RAG 管线**（这是测试重点）：
   - 上传一份参考文档（PDF / Word / Markdown）
   - 观察是否自动转换为 Markdown → 分块 → 索引（状态条会推进）
   - 索引完成后，去"知识"或"头脑风暴"里提问，确认能检索到该文档内容
4. **验证写作**：新建写作 → 生成大纲 → 逐段生成 → 导出。

### 4. 日常使用

- **关闭窗口**不会退出应用，会最小化到**系统托盘**（右下角 Synthetix 图标）。
- 托盘右键 → **Open Synthetix** 重新打开窗口；**Quit** 才真正退出（会干净停止后台服务）。

## 已知限制（本次测试版）

- ❌ **仅 Windows**：macOS 后续再做。
- ❌ **无自动更新**：后续版本需手动重装。
- ⚠️ **安装包较大（~600MB）**：因为捆绑了完整的 Python 运行时（docling / sentence-transformers / onnxruntime 等 RAG 依赖）。后续会优化（模型权重按需下载）。
- ⚠️ **应用图标为默认**：暂用 Electron 默认图标，品牌图标后续补。
- ⚠️ **首启需要联网**：首次启动会下载 LiteLLM 模型目录（~1.5MB，用于模型参数自动填充）和 sentence-transformers 嵌入模型（首次用 RAG 时，约几百 MB）。下载失败不影响应用启动（catalog 降级为空），但嵌入模型下载失败会无法索引文档。
- ⚠️ **需自备 LLM API**：应用本身不含模型，需在设置里配置你自己的 LLM provider（OpenAI / DeepSeek / Ollama 等）。

## 出问题怎么办

### 应用打不开 / 闪退
查看日志：`%APPDATA%\Synthetix\server.log`（后台服务日志）。
把这个文件发给我。

### 想彻底重置（清数据）
退出应用 → 删除整个 `%APPDATA%\Synthetix\` 文件夹 → 重新启动。会重新走首启初始化。

### 端口冲突
应用默认用 3000 端口，被占用会自动换 3001+。若仍有问题，退出其他占用 3000 的程序（如其他本地开发服务）。

## 技术信息（给开发者）

- 架构：Electron 壳 + 后台 spawn Next.js 服务 + 捆绑 Python RAG worker
- 构建命令：`node scripts/build-electron.mjs`
- 数据目录：`%APPDATA%\Synthetix\`（dev.db、documents、rag、.env）
- 安装目录：`%LOCALAPPDATA%\Programs\Synthetix\`（程序本体，含 resources/app 的完整 Next + Python bundle）
- 详细方案：`docs/desktop-packaging-distribution-plan-2026-06-29.md`
