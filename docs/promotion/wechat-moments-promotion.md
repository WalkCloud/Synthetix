## 推荐主文案

这段时间，我把自己一直想做的一个工具正式开源了，名字叫 Synthetix。

做售前、方案和项目材料时，经常会碰到类似的情况：客户资料、技术文档、历史方案和会议材料一大堆。真正费时间的，不只是写，而是先把资料读明白，理清结构，写完以后还能回答“这一段的依据在哪里”。

普通 AI 对话处理少量内容很方便，但资料一多，文件很难一次放全；长文写到后面，前后也容易对不上。于是我做了一个面向大量文档的自托管 AI 知识与长文写作工作台。

它可以把 PDF、Word、PPT、EPUB、Markdown 等资料整理成可搜索的知识库，生成知识 Wiki 和关系图谱。写作时先梳理需求、生成大纲，再按章节调用相关资料；每个章节可以查看引用、比较模型结果、保留版本，最后还能用拓扑图检查哪些材料支撑了哪些内容。

这个项目不是想替人做专业判断，也不保证 AI 写出来的内容天然正确。我更希望它能把“读资料、找依据、搭结构、逐章写”这套过程收拢到一个工作台里，让人有地方检查和修改。

目前 Synthetix 已按 Apache 2.0 协议开源，还在持续完善。如果你平时也要处理大量材料，或者对 AI 知识库、RAG、长文写作感兴趣，欢迎帮我看看。觉得方向有用的话，也请到 GitHub 点个 Star。对个人开源项目来说，这是很直接的鼓励。

GitHub：<https://github.com/WalkCloud/Synthetix>

## 常见回复模板

### 这是做什么的？

主要是把大量文档整理成可搜索、可复用的知识库，再基于这些资料做大纲和逐章节写作。生成内容可以查看来源，也能检查整篇文档用了哪些材料。

### 和直接把文件发给 AI 有什么区别？

直接发文件适合一次性、资料量不大的任务。Synthetix 会长期保存和索引资料，把检索、Wiki、图谱、大纲、章节、引用和版本串起来，更偏向多资料、多章节的持续工作。

### 能否离线使用？

项目支持自托管，也可以连接 Ollama 等本地服务。能否完全离线取决于模型和 Embedding 等 Provider；如果使用远程模型，对应任务内容会发送给远程服务。

### 怎么安装？

GitHub Releases 提供 Windows 安装方式，也可以按 README 从源码启动。第一次使用还需要配置聊天模型和 Embedding 模型。

### 如何支持？

最简单的方式是去 GitHub 点个 Star。愿意实际试用、提 Issue、补文档或参与贡献，就更感谢了。

### 可以用于客户项目吗？

项目使用 Apache 2.0 协议，可以按许可证要求使用。涉及客户资料时，仍然要结合公司的安全规范选择部署方式和模型 Provider，敏感内容不要发送到未经批准的外部服务。
