## 45–60 秒口播稿

如果你试过把几十份 PDF、项目材料或者技术规范交给 AI，再让它写一篇完整报告，大概率遇到过这几个问题：资料塞不完，长文写着写着前后对不上，最后连一句话来自哪份文件都说不清。

所以我做了 Synthetix，一个面向大量文档的自托管 AI 知识与长文写作工作台。

你可以把 PDF、Word、PPT、EPUB 和 Markdown 等资料放进去。系统会把它们整理成可搜索的知识库，还能生成知识 Wiki 和关系图谱。

写作时，它会先陪你梳理需求、生成大纲，再按章节调用相关资料。每个章节都可以查看引用来源、比较不同模型的结果，也能通过拓扑图检查整篇文档用了哪些材料。

项目已经用 Apache 2.0 协议开源。它还在持续完善，如果这个方向对你有用，欢迎去 GitHub 看看，顺手点个 Star。对开源项目来说，这就是很实在的支持。

## 发布简介

做报告、技术方案或研究材料时，资料太多、长文不连贯、引用找不到，是我一直想解决的问题。

于是我做了 Synthetix：把大量文档整理成知识库，再基于这些资料进行头脑风暴、大纲生成、逐章写作和来源检查。

项目已按 Apache 2.0 协议开源。欢迎体验、提建议，也欢迎去 GitHub 点个 Star。

## 评论区回复模板

### 有安装包吗？

有 Windows 安装方式，也可以从源码运行。具体步骤都写在 GitHub README 里，建议先看一下前置条件和模型配置说明。

### 能完全离线吗？

可以接 Ollama 等本地服务。是否完全离线取决于你配置的模型、Embedding 等 Provider；如果接远程服务，相关任务内容会发送到对应服务处理。

### 和普通知识库有什么区别？

它除了检索资料，还把头脑风暴、大纲、逐章写作、引用、版本和拓扑检查串成了一套长文工作流。更适合报告、方案和多章节文档，而不是只问一个问题。
