## 可直接发布的正文

最近两年我越来越常遇到一种工作：手上有几十份 PDF、Word、PPT、技术规范和历史方案，需要先读懂，再整理成一篇结构完整的报告。

普通 AI 对话在这种场景里挺容易卡住。

文件不可能一次全塞进去；对话拉长之后，它会忘记前面讲过什么；文档写出来看着很顺，但真要追问“这句话来自哪份资料”，又得重新翻原文件。

我不太想继续靠复制粘贴和一堆临时对话解决这件事，所以做了 Synthetix。

它是一个自托管的 AI 知识与长文写作工作台。你可以把 PDF、DOCX、PPTX、HTML、EPUB、TXT、Markdown 等资料放进去，系统会把内容拆分、索引，并整理成可以反复使用的知识库。

我最想保留下来的，是整理资料的过程。资料问完一次不会丢掉，除了关键词和语义搜索，还可以浏览知识 Wiki、实体关系和来源证据。下次写别的内容时，不用重新从一堆文件开始翻。

写长文时，我也不想让 AI 一口气硬写完整篇。Synthetix 会先梳理目标和受众，生成可以修改的大纲，再按章节调用相关资料。最后怎么写仍然需要人来判断，不过结构和上下文会更容易管住。

还有一个很现实的问题：来源。每个章节可以保存引用，也可以用拓扑图查看哪些文档支撑了哪些章节。做研究报告、技术方案或客户材料时，这一点往往比“写得像不像”更重要。

目前还支持相同上下文下的模型 A/B 对比、章节版本管理，以及 Markdown、PDF、DOCX 导出。

它比较适合：

- 经常要读大量资料并写报告、方案或项目文档的人；
- 需要维护私有知识库、又希望能自己部署的团队；
- 对 RAG、知识图谱和 AI 长文工作流感兴趣的开发者。

它也不是万能的。现阶段重点是文本资料和长文生产，不是原样复刻复杂版式，也没有把表格文件列入当前写作管线。是否完全离线，则取决于你使用本地模型还是远程 Provider。

项目已经按 Apache 2.0 协议开源。我还在继续打磨它。如果你也被“大量资料怎么真正用起来”这个问题折腾过，欢迎去 GitHub 看看。暂时没空安装也没关系，点个 Star，方便以后回来，也算给这个项目一点继续做下去的动力。

## 常见评论回复

### 普通用户能用吗？

可以，但第一次使用需要配置模型 Provider。Windows 有安装方式，具体步骤和前置条件可以看 GitHub README。如果平时接触过 API 或 Ollama，上手会更顺一些。

### 能不能用国产模型？

可以配置 OpenAI 兼容端点，也支持 Anthropic、Ollama 等方式。具体能用哪些模型，取决于对应服务是否提供项目需要的聊天或 Embedding 能力。

### 数据会上传吗？

文件和应用数据可以自托管。模型处理是否外发取决于你的 Provider：接远程模型时，任务所需内容会发送给该服务；使用本地服务可以减少外部传输。

### 适合写论文吗？

它适合整理资料、搭结构、逐章写作和检查引用来源，但不应该替代研究判断、事实核验和正式的学术引用管理。
