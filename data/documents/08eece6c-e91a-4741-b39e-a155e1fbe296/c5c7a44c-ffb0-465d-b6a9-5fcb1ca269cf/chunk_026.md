流水线是一个自定义的 CI/CD 流水线模式，定义了包含构建、测试和发布的完整构建过程，它基于 Jenkins 进行业务开发，旨在通过自动化持续交付快速应用到代码管理与集成、部署、发布等环节。

ACP全栈云平台通过集成 Jenkins 以及扩展各种常用的工具链丰富和完善了 CI/CD 体系，通过多种创建流水线的方式整合工具链，从而为用户更方便的搭建出属于自己的流水线，为企业快速实现 DevOps 落地。

* 创建流水线

为了应对企业不同场景对流水线的需求，平台流水线支持模板创建、脚本创建、多分支创建和图形化创建，从多个维度提供便捷的流水线创建服务。

* 图形化创建流水线是通过可视化的界面操作快速创建流水线的一种方式，可以更加直观的看见流水线上的每一个任务，方便地展示出流水线的每个阶段，从而实现创建流水线的视图化、透明化。

目前DevOps 持续交付的流程是完全基于Jenkins实现的，随着云原生技术的发展，现有Jenkins的架构已无法满足各种复杂业务场景，如高可用。为此，平台基于云原生开源框架Tekon为 DevOps 平台重新定义了持续构建流程，以满足在不同项目、不同命名空间下的构建需求。

平台践行 GitOps 的理念，将 YAML 文件进行版本管理，以 Git 作为持续交付的中心，开发人员可以使用熟悉的工具发起拉取请求，根据 YAML 变更自动进行构建任务，以简化 Kubernetes 的应用程序的操作任务。

平台现支持使用简单的YAML文件，定义从代码检出到构建制品的全部流程，并支持手动参数触发和代码仓库触发器自动触发的功能，详情如下：

* 创建构建：支持在项目资源下，不同命名空间中使用 YAML 编写的方式快速定义构建流程。
* 构建详情：支持查看构建基本信息详情以及详细构建步骤信息。
* 执行构建：支持根据代码仓库的 分支、PR、commit 手动执行构建任务，实现自定义执行构建并产出不同的制品，丰富触发场景。
* 触发器：支持使用代码仓库触发器，根据触发事件自动触发构建任务。
* 创建构建

支持在项目资源下，不同命名空间中使用表单或 YAML 编写的方式快速定义构建流程。

管理构建



对于复杂的业务系统，当需要有新的功能开发或者业务更新时，从代码编写、编译、构建、自检、集成验证、类生产验证、上线的各个阶段都需要耗费大量的人力和时间，并且容易受到人为因素影响而出错。持续集成和持续交付由于具有标准化和自动化特点，可以很好的解决资源浪费问题。

持续发布流水线定义了从代码被构建为某一种制品开始后的持续交付过程，基于已有命名空间，用户无需关心底层基础设施的管理与配置，通过一个简单的 YAML 编排文件，就可实现从制品部署到开发环境进行集成验证、测试环境功能测试、类生产环境性能测试、部署到线上环境的整个过程，此外，还可基于已部署的环境，进行一系列的自动化验证、监控等，根据一定的质量门指标实现过程的自动化及可视化，同时支持设置人工审批，减少人为操作产生的风险。
