Envoy 是专为大型现代 SOA（面向服务架构）架构设计的 L7 代理和通信总线，是 Istio 的核心组件之一，以 Sidecar 的方式与服务运行在一起，对服务的流量进行拦截、转发。能够提供流量路由、流量控制等能力。

平台上的 EnvoyFilter 资源是项目下资源，可以被项目下多个服务网格、命名空间中的微服务共享。不同于其他网络对象，EnvoyFilters 是可叠加应用的。在指定的命名空间中，对于指定的计算组件，可以存在任意数量的 EnvoyFilters。

* 创建EnvoyFilter

创建 EnvoyFilter 时，可根据当前项目下服务网格支持的 Istio 版本添加不同适配版本的 EnvoyFilter 资源，使一个 EnvoyFilter 可以适配不同服务网格的 Istio 版本。

进一步将创建的 EnvoyFilter 绑定到微服务后，即可将用户自定义的能力通过 EnvoyFilter 下发到服务网格下的微服务。

微服务网关作为微服务架构的入口，可为微服务平台提供路由转发、API 管理、权限校验过滤等能力。

* 微服务网关的主要功能如下：
* 智能路由：接收外部的一切请求，并转发到后端的对外服务。
* 提示：服务网关只转发外部请求，服务之间的请求不经由网关转发。
* 权限校验：当客户端发起请求时，校验客户端用户的访问权限。
* API 监控：监控经过网关的 API 调用请求，以及网关本身的一些性能指标（例如，GC等）。
* 限流：与API监控配合，进行API级别的限流。
* 微服务网关包括入口网关（istio-ingressgateway）和出口网关（istio-egressgateway）。
* 入口网关：入口网关是 Istio 的重要资源对象之一，定义了所有入站流量流经服务网格的入口点，用于治理网格边缘的入站流量，并对其进行路由编排。
* 出口网关：出口网关是 Istio 的重要资源对象之一，定义了服务网格出站流量的出口点。网格内注册的外部服务可绑定到出口网关，使得对外访问流量从出口网关定点发出。



* 创建网关入口

网关入口即服务网格外部流量进入服务网格的入口。基于网关入口进行路由管理和策略设置，可将进入服务网格的流量根据路由规则，定向转发至平台上已有的 API 分组、微服务，实现服务网格入口流量的监控和路由。
