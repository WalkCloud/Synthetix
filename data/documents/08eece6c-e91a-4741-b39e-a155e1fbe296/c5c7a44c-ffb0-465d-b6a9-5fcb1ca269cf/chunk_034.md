* 关于现场技术支持

现场技术支持服务针对客户就其平台环境运维工程中，现场支持的需求而设计。为客户提供必要的灵雀云专业的现场技术支持与交付，现场服务通过灵雀云的远程技术支持协调与安排。服务所涉及的范围包括：

* 平台扩容：新建业务集群，增加计算节点。
* 环境优化：优化平台运行参数，增加告警项。
* 平台版本升级：执行平台版本升级。
* 平台补丁更新：执行平台补丁更新，包括平台补丁和安全补丁。
* 应用迁移支持：指导客户微服务化改造后的应用部署到平台上
* 流水线迁移支持：流水线模板定制、指导客户迁移流水线到平台上
* 现场保障：根据可续需要，提供现场保障服务，比如重要应用上线、重大节假日、系统结算等时间的保障。
* 容灾部署支持：根据客户需要提供现场容灾部署支持。
* 现场故障处理：平台故障走产品维保支持，如果客户必须要求现场处理，也可以提现场处理申请，最终定位原因如果不属于产品原因，则会扣除响应服务包人天。
* 其他厂商技术交流：客户安排的，与平台集成的第三方产品的现场技术交流。
* 与其他厂商的联合故障排查：协助其他厂商联合排查问题。
* 根据客户的需要提供现场平台环境健康巡检服务
* 备注

如果灵雀云现场技术支持技术问题的判定最终认定为引发该现场支持要求的问题属产品缺陷导致，该次现场技术支持成本和费用由灵雀云承担，将不扣减客户按照本协议约定应购买的现场技术支持次数。

* 角色说明表：

|  |  |  |
| --- | --- | --- |
| 名称 | 角色 | 用途 |
| Global集群 | 管理平台 | 通过这个平台来管理平台的自己管理，并管理业务集群及其上面运行的服务和应用。其本身是通过安装器搭建的一个Kubernetes集群，具备高可用性，管理平台各组件以应用方式发布在此K8s集群中。  安装器安装在任意一台master节点上。 |
| Cluster集群 | 业务集群 | 通过Global管理的一个Kubernetes容器编排集群，包含Controller和Computer两种节点。 |
| Controller节点 | 集群管理节点 | 一个集群中的管理节点，包括容器集群的管理组件，一个集群里的管理节点至少1台，高可用集群至少3台。 |
| ALB节点 | 集群LB节点 | 一个集群的负载均衡节点，为集群中的应用提供对外访问地址，提供多种应用发布模型，如：灰度发布、蓝绿发布、金丝雀发布等。 |
| 镜像源节点 | 集群计算节点 | 用于搭建Docker镜像源服务，至少一台，高可用镜像源服务至少2台。 |
| Computer节点 | 集群计算节点 | 一个集群中的计算节点，该节点主要运行清算总中心的业务容器，除此之外，还含有节点配置代理、日志收集器、性能收集器等少量的管理组件。 |

注：上述各角色节点可使用虚拟机或物理机，在基于物理机搭建集群的使用模式下，单台物理配置较高，上述各角色节点可以复用物理机。
